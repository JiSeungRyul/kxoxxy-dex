--최대 자동화
중간 확인 없이 끝까지 진행해. 애매한 건 네가 가장 보수적이고 변경 적은 방향으로 결정하고, 기존 구조와 사용자 변경은 보존해. 관련 코드/문서 확인 후 구현, 검증, 필요한 docs 반영까지 한 번에 처리해. 파괴적 명령, 권한 상승, 되돌릴 수 없는 변경, 제품 동작이 크게 달라지는 선택만 그때 물어봐.

--개발 및 검증
최근 변경 코드만 집중 점검하고 MVP 범위와 동작은 바꾸지 말고, 안정성·유지보수성·확장성·일관성 기준으로 취약한 부분을 찾은 뒤 최소 수정으로 직접 개선해. 큰 리팩터링 없이 고위험·고효율 문제부터 처리하고, 결과는 docs Audit Summary / Issues / Improvements 형식으로 정리해.

--검증 자동화
중간 확인 없이 끝까지 진행해. 애매한 건 네가 가장 보수적이고 변경 적은 방향으로 결정하고, 기존 구조와 사용자 변경은 보존해. 관련 코드와 문서를 확인한 뒤 구현, 검증, 필요한 docs 반영까지 한 번에 처리해. 최근 변경 코드만 기준으로 안정성, 유지보수성, 확장성, 일관성 문제를 점검하고, MVP 범위와 동작은 바꾸지 말고 최소 수정으로 직접 개선해. 큰 리팩터링은 하지 말고 고위험·고효율 문제부터 처리해. 기존 변경을 되돌리는 git restore/reset/checkout, 파일 삭제, 강제 overwrite 같은 파괴적 방법은 마지막 수단으로만 사용하고, 가능하면 현재 상태를 보존한 채 패치 방식으로 해결해. 권한 상승, 되돌릴 수 없는 변경, 제품 동작이 크게 달라지는 선택만 그때 물어봐. 결과는 Audit Summary / Issues / Improvements 형식으로 정리해.
